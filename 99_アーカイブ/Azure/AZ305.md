### Azure Blueprints を考慮して設計する
* Azureブループリントを使用すると、ブループリント定義（展開する必要があるもの）とブループリント割り当て（展開されるもの）の間の関係が保持されます。
  + この接続は、展開の追跡と監査の改善をサポートします。 
  + Azureブループリントは、同じブループリントによって管理される複数のサブスクリプションを一度にアップグレードすることもできます。
* 通常、サブスクリプションで適切なAzureロールベースのアクセス制御（Azure RBAC）を使用しているユーザー（「所有者」ロールなど）が、任意のリソースを変更または削除できるようにすることができます。
  + このアクセスは、AzureBlueprintsがデプロイされた割り当ての一部としてロックを適用する場合には当てはまりません。
  + 割り当てが[読み取り専用]または[削除しない]オプションで設定されている場合、サブスクリプションの所有者でさえ、保護されたリソースに対してブロックされたアクションを実行できません。
  + サブスクリプションレベルでリソースロックを使用しようとすると、所有者は削除できます。
* ルート管理グループ

    - このルート管理グループを使用すると、グローバルポリシーとAzureの役割の割り当てをディレクトリレベルで適用できます。
  + ブループリントの各公開バージョンは、既存の管理グループまたはサブスクリプションに割り当てることができます。
  + ブループリント定義を作成するときは、ブループリントを保存する場所を定義します。
  + ブループリントは、所有している管理グループまたはサブスクリプションに保存できます
  + 寄稿者のアクセス。場所が管理グループの場合、ブループリントを使用して、その管理グループの子サブスクリプションに割り当てることができます。

### ID およびアクセス管理 (IAM) を設計する

  + ロールは、Active Directoryのユーザー、グループ、サービスプリンシパル、およびマネージドIDに適用されます。
  + 個人およびグループには、組み込みの役割またはカスタムの役割を使用できます。
### Azure Active Directory について設計する
* （アプリケーションプロキシ）でコネクタをダウンロードし、（エンタープライズアプリケーション）で新しいアプリケーションを作成する必要があります。
  + 事前認証オプションでは、[パススルー]または[Azure Active Directory]を選択でき、どちらも機能します。
  + 「AzureActiveDirectory」を使用して、条件付きアクセスとMFAを利用できるようにする方を奨励します。
* AzureADアプリの登録
  + Azure Active Directory（AD）は、クラウドベースのディレクトリおよびID管理サービスを提供します。
  + azureADを使用して、アプリケーションのユーザーを管理し、azureActiveDirectoryを使用してアプリケーションへのアクセスを認証できます。アプリケーションをAzureアクティブディレクトリテナントに登録します。
### Azure Active Directory B2B (企業間) を設計する

  + ゲストユーザーを招待し、特権ID管理でAzureリソースロールを割り当てます
  + Azure Active Directory（Azure AD）ゲストユーザーは、Azure AD内の企業間（B2B）コラボレーション機能の一部であるため、外部のゲストユーザーとベンダーをAzureADのゲストとして管理できます。
* Azure Active Directory B2B の機能を使用して、外部のゲスト ユーザーと共同作業を行うことができます。

### 条件付きアクセスについて設計する
* 条件付きアクセスポリシー
  + 条件付きアクセスポリシーを使用すると、必要に応じて適切なアクセス制御を適用して、組織を安全に保ち、不要なときにユーザーの邪魔にならないようにすることができます。
### Identity Protection について設計する
* Privileged Identity Management
  + Privileged Identity Management は、時間ベースおよび承認ベースのロール アクティベーションを提供し、関心のあるリソースに対する過度の、不要な、または誤用されたアクセス許可のリスクを軽減します。 Privileged Identity Management の主な機能以下です。
    - Azure AD および Azure リソースへのジャスト イン タイムの特権アクセスを提供する
    - 開始日と終了日を使用してリソースへの期限付きアクセスを割り当てる
    - 特権ロールをアクティブ化するには承認が必要
    - 多要素認証を適用して任意のロールをアクティブ化
    - 正当な理由を使用して、ユーザーがアクティブ化する理由を理解する
    - 特権ロールがアクティブ化されたときに通知を受け取る
    - アクセス レビューを実施して、ユーザーがまだロールを必要としていることを確認します
    - 内部監査または外部監査の監査履歴をダウンロードする
    - 最後のアクティブなグローバル管理者ロールの割り当てが削除されないようにします
### アクセス レビューについて設計する
* Azure Active Directory (Azure AD) を使用すると、ユーザーに適切なアクセス権を確実かつ容易に設定できます。 
  + ユーザー自身または意思決定者に対し、アクセス レビューに参加してユーザーのアクセス権を再確認 (証明) するよう求めることができます。 
  + レビュー担当者は、Azure AD からの提案に基づいて、各ユーザーの継続的なアクセスのニーズを評価できます。 
  + アクセス レビューが完了したら、変更を加え、不要になったアクセス権をユーザーから削除できます。
### マネージド ID を設計する
* AzureリソースのマネージドIDは、AzureActiveDirectoryの機能です。
* ユーザーが割り当てた管理対象IDを共有できます。同じユーザーが割り当てたマネージIDを複数のAzureリソースに関連付けることができます。
* システム割り当てマネージド ID は共有できません。単一の Azure リソースにのみ関連付けることができます。

### Azure Key Vault について設計します
* テンプレートのデプロイ中にキーボールトにアクセスするには、キーボールトのenabledForTemplateDeploymentをtrueに設定します。
* テンプレートを展開するユーザーは、リソースグループとキーボールトのスコープに対するMicrosoft.KeyVault / vaults / deploy/action権限を持っている必要があります。
* 独自の暗号化キーを使用して、ストレージアカウントのデータを保護できます。 
  + 顧客管理キーを指定すると、そのキーは、データを暗号化するキーへのアクセスを保護および制御するために使用されます。
  + 顧客が管理するキーを格納するには、AzureKeyVaultまたはAzureKeyVault管理対象ハードウェアセキュリティモデル（HSM）（プレビュー）のいずれかを使用する必要があります。
* シークレット、キー、証明書などのキーボールトオブジェクトをバックアップすると、バックアップ操作によってオブジェクトが暗号化されたBLOBとしてダウンロードされます。 
  + このBLOBは、Azureの外部で復号化することはできません。 
  + このBLOBから使用可能なデータを取得するには、同じAzureサブスクリプションとAzure地域内の主要なボールトにBLOBを復元する必要があります。

### Azure Monitor ログ (Log Analytics) ワークスペースの設計
* Log Analyticsワークスペースは、さまざまなリソースやクエリ/アクションからすべてのログを収集できる1つの中心的な場所です。
* Log Analyticsは、AzureMonitorLogsのデータを使用してログクエリを編集および実行するために使用されるAzureポータルのツールです。 
  + レコードのセットを返す単純なクエリを記述し、Log Analyticsの機能を使用してそれらを並べ替え、フィルタリング、分析することができます。 
  + または、より高度なクエリを記述して統計分析を実行し、結果をグラフで視覚化して特定の傾向を特定することもできます。
* Azure Monitor Log（Log Analyticsによって分析されます）-アプリインサイトは、データ収集の観点からAzureSQLサーバーに対応しません。 
  + App Insightは、アプリとSQLの間の関係マップを提供できますが、実際のデータは提供されません。 
  + SQLのエントリポイントは、Analyticsにデータを提供するAzureMonitorになります。
* AzureMonitorの機能であるApplicationInsightsは、開発者およびDevOpsプロフェッショナル向けの拡張可能なアプリケーションパフォーマンス管理（APM）サービスです。
  + ライブアプリケーションを監視するために使用します。
  + パフォーマンスの異常を自動的に検出し、問題を診断し、ユーザーがアプリで実際に何をしているのかを理解するのに役立つ強力な分析ツールが含まれています。
* Azure Monitor エージェントをインストールする
  + 次のことが必要な場合は、Azure Monitor エージェントを使用します。 
    - Azure、他のクラウド、またはオンプレミスの任意のマシンからゲスト ログとメトリックを収集する。

  + これらのアカウントは、ブロック BLOB、ページ BLOB、ファイル、およびキューへのアクセスを提供します。 Azure Storage を使用するほとんどのシナリオで推奨されます。* ライフサイクル管理ポリシーは、汎用v2、プレミアムブロックブロブ、およびブロブストレージアカウントのブロックブロブと追加ブロブでサポートされています。

  + 詳細設定では、階層名前空間を変更して、アクセス制御リスト（ACL）ファイルレベルのアクセス許可を有効にすることができます

### Azure Blob Storage のために設計する
* BlockBlobStorageは、BlockBlobおよび追加Blob用のプレミアムストレージアカウントタイプです。
  + トランザクション率が高いもの、またはより小さなオブジェクトを使用するもの、または一貫して低いストレージレイテンシを必要とするものに推奨されます。
* アーカイブ層は、めったにアクセスされないBLOBデータを格納するためのオフライン層です。
  + アーカイブ層はストレージコストが最も低くなりますが、オンライン層（ホットおよびクール）と比較してデータ取得コストと遅延が高くなります。
  + データは、少なくとも180日間アーカイブ層にとどまるか、早期削除料金の対象となる必要があります。
### Azure Files のために設計する
* FileStorageストレージアカウントを使用すると、Azureファイル共有をプレミアム/ソリッドステートディスクベース（SSDベース）のハードウェアに展開できます。
* Azure Filesは、オンプレミスのActive Directoryドメインサービス（AD DS）とAzure Active Directoryドメインサービス（Azure AD DS）の2種類のドメインサービスを通じて、サーバーメッセージブロック（SMB）を介したIDベースの認証をサポートします
* Azure File Sync
