### 工事進捗表示アプリ開発プロジェクト
#### 概要
　　工事進捗管理システムにて、登録された工事進捗をスマートフォンのアプリで表示する。
　　アプリはカメラでQRコードを読み取り、登録された工事進捗を呼び出して表示することが主な機能となる。
　　２０１９年９月時点、アプリはリリース済みで現在はGPSとNFCを利用する追加機能を開発中。
#### プロジェクト
* メンバー
    + プロジェクトマネージャ１名と開発要員３名
        - リリース後は開発要員1名
* 役割
    + フロントエンド開発要員へ指示する開発リーダを担当
    + 主にバックエンドのWebAPI実装とインフラストラクチャを担当
    + リリース後は、追加機能の開発を担当
* 目標
　
#### 課題解決
　　スマートフォンの会社での使用が初めてであったため、技術選定から行う必要があり、時間の制約がある中で開発ノウハウを蓄積するのに苦労した。
* 技術選定
    + 社内の開発要員にCordovaを使用したことがある者がいたため、本プロジェクトのフロントエンド開発要員に引っ張り、技術選定をお願いした。
    + 個人的には今後主流となりそうなPWAをやりたかったので、サンプルアプリを作成して技術選定の際の参考にしてもらった。
    + 最終的にはPhoneGapを使用した開発に決めたが、情報が少ないのがその後ネックに
* 第1次開発
    + フロントエンド開発メンバーとスマホアプリ開発について勉強会を行い、ノウハウを蓄積していった。
    + 私はAPI実装が別にあったため、プロマネやユーザの間に入り、フロントエンド開発メンバーのフォロー中心に動いた。
    + WebAPI実装自体は、以前の「工事進捗管理システム開発プロジェクト」で行っていたデータベース中心の開発であるため、苦労はしなかった。
* 第2次開発
    + 第1次開発のリリース後は追加機能作成のため、フロントエンド開発も私が担当することとなった。
    + 目的はGPSとNFCの機能を使用して、よりスマートフォンらしいアプリへと改善すること。
    + GPS機能では、将来的にアプリからデータ登録することを念頭に、ユーザが処理を行った場所を記録する。
    + NFC機能では、NFCタグを読み取り対象データを表示する。
    + PhoneGapプラグインを使用することで、容易に機能実装できるとのことだったが、NFCプラグインが動作しなかった。原因はTypeScriptのバージョンが現行アプリのと合わなかったためだけであったが、PhoneGapの情報が少なかったのもあり、余計な時間がかかった。

### 工事進捗管理システム開発プロジェクト
#### 概要
    製品製造における工程（全工程のことをまとめて工事と呼ぶ）のスケジュールを計画し、その進捗を管理する。スケジュール作成のコア機能に生産スケジューラの「ASPROVA」というツールを採用しており、ユーザがASPROVAのデータ表示や登録に使用する周辺システムや、周辺システムで登録したデータをASPROVAへ流すバッチ、ASPROVAへ機能を追加するプラグインなどを開発する。
#### プロジェクト
* メンバー
    + プロジェクトマネージャ１名と開発要員６名（内協力社員３名）
* 役割
    + 主にデータ分析とフロントエンド（周辺システム）へのデータ提供を担当
* 目標

#### 課題解決
    既存の自社システムのデータベースと情報を合わせる必要があることが課題であった。
* メンバーへの教育
+ 自社の製造工程に係る情報について詳しく知っているのが、私とPMだけだった。
+ そのため、他のメンバーへ教育を行うべく、資料作成と勉強会を実施した。
* 機能別に担当者と連携
+ ASPROVAプラグイン開発担当者とは、ASPROVAのデータ構造と自社システムのデータベースとの連携について協議し、周辺システムのデータベース構築の基本設計を行った。
    - ASPROVAについて互いに知識がないため、ベンダーからの情報を取り入れながら検討を重ねた。おかげでASPROVAに関する知識をわずかながら得ることができた。
+ 周辺システムの開発担当者とは、データベースの基本設計をもとに周辺システムに渡すデータのインターフェースを作成した。
    - 周辺システムに関しては、開発が進むにつれユーザ要望が膨らみ、開発当初考えていたデータ構造から大分逸脱してしまい、ボトルネックになりつつあるのが反省点
* その他
+ ユーザへのデータ提供として、APIやExcelのアウトプットを作成。

### 製造管理システム再開発プロジェクト
#### 概要
    現行の製造管理システムは2002年から2004年にかけて開発され、その後ほぼ改修がされずに運用されていたが、日々の運用負荷が高く、抜本的な改修を行うべくプロジェクトが開始された。

* プロジェクトは開発要員が私だけで他案件と輻輳していたこともあり、5年掛かっている。
#### プロジェクト
* メンバー
    + プロジェクトマネージャ１名と開発要員１名
* 役割
    + 開発全般を担当
#### 課題解決
    改修対象となる製造管理システムは全8件あり、それぞれの製造部門が主管となっている。
    各製造部門に対し現状の課題を挙げてもらい、できる限り要望に沿う形でかつ運用保守もしやすくなるように改修を行った。
* 保守切れソフトウェアの変更
    + HirDB を PostgreSQL へ
    + JP1 を Hinemos へ
* 技術的負債の解消
    + CやC++ を VB.NET へ
    + ASP を ASP.NET MVC へ
    + 開発環境の変更を行いつつ、現行の機能は損なわず、またリファクタリングも極力しやすく改修を行った。
* 利便性の向上

* 運用方法の見直し
